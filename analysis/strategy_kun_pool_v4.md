## 2. 入池逻辑详解

### 入池

**触发**: 个股 d_trend 上一日 ≥ 11 **且** 当日 < 11
**业务含义**: 个股趋势线从中部以上跌破 11, 表示已进入"深度回调区"

### 在池

**状态**: 入池后, 持续保持"在池"状态 (无论 trend 之后涨跌)
**作用**: 标记该股"曾经触底", 是合格的反转候选

### 出池

**触发**: 入池后, 当 **大盘 y_gua=坤 + 个股 d_gua=巽 (基础事件)** 成立的当天
**含义**:
- 不论是否真买入 (满仓时也消耗池机会)
- 不论是否通过 score 过滤
- "Gate 1+2 满足" = 出池

### 重新入池

出池后, 该股需要再次 d_trend 下穿 11 才能重新入池.

### 卖点逻辑

```python
def sell_kun_pool_v4(buy_idx, td_seg, max_end):
    """坤+入池 v4 卖点 (bull + TS20)"""
    cross_count = 0
    running_max = td_seg[buy_idx]

    for k in range(buy_idx + 1, max_end + 1):
        if not isnan(td_seg[k]):
            running_max = max(running_max, td_seg[k])

        # 主路: 第 2 次下穿 89
        if running_max >= 89 and td_seg[k] < 89 and td_seg[k-1] >= 89:
            cross_count += 1
            if cross_count == 2:
                return k, 'bull_2nd'

        # 保险: 持仓 ≥ 20 日且 trend 从未到过 89 → 强卖
        if k - buy_idx >= 20:
            valid_max = max(td_seg[buy_idx:k+1])
            if valid_max < 89:
                return k, 'ts20'

    # 兜底: 60 日
    return max_end, 'timeout'
```

## 7. 入池停留时间分布 (业务洞察)

| 入池→触发停留 | n | 胜率 | 30日 ret |
|---|---|---|---|
| 当日 (跨 11 即触发) | 4 | 0% | -5.10% (噪音) |
| 1-5 日 | 8,373 | 56.5% | +4.17% |
| **6-10 日 ⭐** | **8,961** | **62.6%** | **+5.86%** |
| 11-20 日 | 4,588 | 58.4% | +4.51% |
| 21-60 日 | 4,429 | 53.0% | +4.56% |
| **>60 日 ⭐** | **1,531** | **59.1%** | **+8.45%** |

**业务**:
- **6-10 日**: 触底后稳定 1-2 周再反转启动 = 主力悄悄建仓的甜蜜区
- **>60 日**: 触底后长期低位震荡, 突然爆发 = 浮筹清洗充分的强势启动
- **当日**: 跨 11 当日即巽日, 太急不稳 (4 笔 -5.1% 是噪音)

可以加规则: 入池停留 ≥ 1 日才有效 (排除当日触发).

## 11. 实施伪代码

```python
def kun_pool_buy_signal(stock, day, market):
    """v4 买点判断 (5 道关卡)"""
    # Gate 0: 入池 (持续维护)
    # 在外层维护 stock.in_pool 状态
    if not stock.in_pool: return None

    # Gate 1: regime
    if market.y_gua(day) != '000': return None

    # Gate 2: 时机 (出池)
    if stock.d_gua(day) != '011': return None

    # 出池 (无论后续是否买入)
    stock.in_pool = False  # 由调用方维护

    # Gate 3: 强避雷
    if stock.y_gua(day) == '011': return None
    if stock.m_gua(day) == '111': return None

    # Gate 4: score (4 项)
    score = 0
    if market.m_gua(day) == '100': score += 1
    if market.d_gua(day) == '011': score += 1
    if market.m_gua(day) == '010': score += 1
    if stock.m_gua(day) == '010': score += 1
    if score < 2: return None

    return {
        'code': stock.code, 'date': day,
        'price': stock.close(day), 'score': score,
        # score=3 时仓位加倍 (12% 命中, +14.15% 期望)
        'weight': 2.0 if score >= 3 else 1.0,
    }


def kun_pool_pool_update(stock, day):
    """每日更新入池状态 (在 buy_signal 之外调用)"""
    # 下穿 11 时入池
    if stock.d_trend_prev(day) >= 11 and stock.d_trend(day) < 11:
        stock.in_pool = True
```
