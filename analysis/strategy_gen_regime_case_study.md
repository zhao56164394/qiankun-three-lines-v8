## 1. 历史快照: 艮 regime 是什么?

### 大盘卦象定义

```
艮 = y_pos=0 (长期低) + y_spd=0 (中期低) + y_acc=1 (短期高)

含义: "长期还熊, 中期没启, 短期亢奋"
     — 反弹假动作特征, 不是真反转
```

### 历史出现窗口

```
2018-2026 全样本期, 艮 regime 仅出现 2 次:
  w2_2019: 2019 年内, 共 ~5 个交易日 (大熊后期试探)
  w4_2021: 2021-05-26 ~ 2021-06-16, 共 7 个交易日 (抱团末期)

总样本: 4,870 巽日事件, 远小于坤 regime 155K
```

### 段背景

| 段 | 时期 | 大盘环境 | 巽日 baseline |
|---|---|---|---|
| w2_2019 | 2019 短暂艮窗口 | 19 年初反弹刚到位, 短期亢奋透支 | **-4.63%** (大熊段, 巽日反而亏) |
| w4_2021 | 2021-05/06 抱团末段 | 沪指 3500-3600, 新能源/半导体启动 | **+7.12%** (抱团股巽日机会) |

## 5. 卖点 — 状态机 td80 → m_dui (**v3 定稿**)

### 卖点研究路径 — 从过拟合到真策略

```
v1: 描述性 case study, 无具体卖点
  ↓
v2: bull (第 2 次穿 89), 期望 +13.67%, 持仓 57 日
  ↓ 持仓诊断发现 87% timeout, d_trend 信号在艮上失效
v2.1: 试 fix80 (固定 80 日), 期望 +28.86%
  ❌ 但 80 日是从历史曲线峰值反推, 完全是过拟合
       下次艮 regime 出现可能 60 / 100 日, magic number 就死了
v3: 状态机 td80 → m_dui ★ 定稿
       不依赖任何固定天数, 完全是状态信号
```

### v3 卖点逻辑 (无 magic number)

```python
def sell_gen_regime_v3(td_seg, gua_seg, stk_m_seg):
    """艮 regime case study v3 卖点 - 状态机
    
    阶段 1 (进入态): d_trend 到过 ≥80 → 进入"主升状态确认"
    阶段 2 (退出): 进入态后, 个股月卦切换到 110 兑 → 卖
    """
    in_state = False
    for k in range(len(td_seg)):
        if not in_state:
            # 进入态判定
            if td_seg[k] >= 80:
                in_state = True
            continue
        # 退出: 月卦兑 (中期顶信号)
        if stk_m_seg[k] == '110':
            return k, 'm_dui'
    return None, 'no_signal'
```

### 业务逻辑

```
阶段 1 进入态 (td80):
  d_trend ≥ 80 = 个股已实际进入"强势主升状态"
  这不是预测, 是确认 — 已经在主升才进入态

阶段 2 退出 (m_dui):
  个股月卦切到 110 兑 (天位低 + 中位高 + 地位高)
  含义: 月线层面出现"中期顶"信号
       中期已经过热, 该走了
  
两阶段连起来: "确认进入主升 → 等到月线见顶 → 卖"
完全是个股自身的状态演化, 不依赖固定时间
```

### v3 核心优势 (vs 过拟合的 fix80)

1. **没有任何 magic number**: 80 (td80) 是公认的 d_trend 强势阈值, 不是从艮历史反推的; m_dui 是月卦中期顶信号, 是项目通用业务定义
2. **状态可解释**: "已进入主升 → 月线见顶" — 任何人能用业务理解
3. **可推广性**: 这套状态机在坤 / 坎 / 其他 regime 上也能直接套, 不需要重新优化天数
4. **timeout 是好事**: timeout 期望 +49.60% 说明"信号没出 = 主升还在", 这个无信号本身是有效的

| 退出信号 | 第 2 次下穿 89 (穿89-回-穿89) | 个股月卦切到兑 (中期顶) |
| 假突破处理 | TS20 时间止损 | 不需要 (假突破跌不深) |
| 设计哲学 | "等到信号反复确认" | "等到中期见顶" |
| 业务故事 | 反弹的探索 → 反复测试阻力 | 亢奋持续爆发 → 月线见顶 |
